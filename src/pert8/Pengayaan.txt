Setelah anda menyelesaikan kode, tolong jawab pertanyaan dibawah ini ya =


1. Apa tujuan utama adanya interface dalam pemrograman berorientasi objek, dan bagaimana interface membantu menciptakan kontrak perilaku?
2. Mengapa interface dapat memiliki banyak implementasi, dan apa makna konseptual dari kemampuan tersebut dalam desain sistem?
3. Apa perbedaan mendasar antara extends dan implements dalam konteks interface, jika dilihat secara konsep tanpa melihat contoh kode?
4. Apa alasan diperkenalkannya default method dalam interface, dan bagaimana fitur ini memengaruhi keberlangsungan (compatibility) suatu sistem yang sudah berjalan?
5. Bagaimana konsep multiple interface memperluas fleksibilitas desain tanpa menimbulkan masalah kompleksitas yang biasa muncul pada multiple inheritance?

jawaban:
1. Interface berfungsi sebagai kontrak standar yang mewajibkan class-class tertentu memiliki method-method yang sama, meskipun implementasi detailnya bisa berbeda. Ini menjaga konsistensi dan memudahkan fleksibilitas pengembangan sistem karena class-class berbeda dapat digunakan dengan cara yang seragam.
2. Karena interface hanya menetapkan aturan perilaku tanpa menentukan cara penerapannya, satu interface dapat diwujudkan oleh berbagai class dengan logika yang berbeda. Hal ini membuat sistem mudah dikembangkan: class baru dapat ditambahkan tanpa mengganggu kode lama, asalkan mengikuti kontrak interface yang sama.
3. Extends digunakan untuk mewarisi sifat dan kode dari class lain (hubungan "adalah"), sedangkan implements adalah janji untuk memenuhi kontrak perilaku dari interface (hubungan "dapat melakukan").
4. Default method memungkinkan penambahan fungsi baru di interface tanpa memaksa class lama untuk meng-update kode. Dengan memberikan implementasi dasar, interface dapat berkembang tanpa merusak kompatibilitas sistem yang sudah berjalan, sehingga program lama tetap berfungsi tanpa perubahan.
